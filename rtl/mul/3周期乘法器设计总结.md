## 🎯 设计特点

### 1. 3周期流水线架构

```
周期1: Booth编码 + 第一层4:2压缩 (4个并行)
       ↓ [寄存器1: 620位]
周期2: 第二层4:2压缩 (2个并行) + 第三层4:2压缩 (1个)
       ↓ [寄存器2: 206位]
周期3: 全加器 + 最终加法
```

### 2. 完整的控制接口

**输入：**
- `clk`, `rst_n` - 时钟和复位
- `mul_a_i`, `mul_b_i` - 32位操作数
- `mul_a_sign`, `mul_b_sign` - 符号控制
- `mul_start` - 启动信号

**输出：**
- `mul_result` - 64位结果
- `mul_done` - 完成信号
- `mul_busy` - 忙碌信号

### 3. 状态机控制

```
IDLE → CYCLE1 → CYCLE2 → CYCLE3 → IDLE
```

## 🔧 集成到CPU的方法

### 方法1：流水线停顿（推荐）

```verilog
// 检测乘法指令
if (is_mul_instruction) begin
    mul_start <= 1'b1;
    stall_pipeline <= 1'b1;  // 停顿IF和ID阶段
end

// 等待完成
if (mul_done) begin
    stall_pipeline <= 1'b0;  // 解除停顿
    // 使用 mul_result
end
```

**优点：**
- ✅ 实现简单
- ✅ 不需要复杂的依赖管理
- ✅ 适合教学和毕业设计

### 方法2：独立乘法单元

将乘法器作为独立功能单元，与主流水线并行运行。

**优点：**
- ✅ 不阻塞主流水线
- ✅ 性能更高
- ✅ 工业界常用

**缺点：**
- ❌ 需要记分牌或Tomasulo算法
- ❌ 实现复杂度高

## 📝 关键设计要点

### 1. 流水线寄存器

**寄存器1（CYCLE1 → CYCLE2）：**
- 保存第一层压缩的8个中间值
- 保存pp17（需要传递到最后）
- 总计：620位

**寄存器2（CYCLE2 → CYCLE3）：**
- 保存第三层压缩的2个中间值
- 保存pp17
- 总计：206位

### 2. 状态机

- 使用2位状态寄存器
- 4个状态：IDLE, CYCLE1, CYCLE2, CYCLE3
- 自动转移，无需额外控制

### 3. 启动和完成信号

- `mul_start`：单周期脉冲，启动乘法
- `mul_done`：单周期脉冲，指示完成
- `mul_busy`：电平信号，指示忙碌状态
