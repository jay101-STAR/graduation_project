## Notes

1. **Little Endian**: RISC-V uses little-endian byte ordering
   - SW 0x12345678 stores as: [78, 56, 34, 12] in memory

2. **Unaligned Addressing**: The test verifies that:
   - SW/LW work correctly at byte offsets 1, 2, 3
   - SH/LH/LHU work correctly at odd byte offsets (1, 3, 5, 7...)
   - SB/LB/LBU work at any byte offset (no alignment requirement)

3. **Sign Extension**: LH and LB sign-extend the loaded value
   - LH: loads 16 bits and sign-extends to 32 bits
   - LB: loads 8 bits and sign-extends to 32 bits
   - LHU/LBU: zero-extend instead of sign-extending

4. **Overlapping Writes**: Tests verify that unaligned writes correctly
   modify only the intended bytes, preserving adjacent data

5. **Error Reporting**: If any test fails, x11 (error counter) is incremented
   and tohost = (error_count << 1) to indicate which test(s) failed
